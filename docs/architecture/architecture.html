Temperance Engine A public, backup-first installer for the author's live PAI + GSD + superpowers + CodeGraph + voice runtime

Business Context

This is a productization, not a fresh design. Temperance Engine did not start from a blank architecture. It is a generalized, path-agnostic, backup-first extraction of the author's own daily-driver Personal AI Infrastructure (PAI) workflow: the 7-phase Observe→Think→Plan→Build→Execute→Verify→Learn loop, GSD's phase-planning/execution lifecycle, superpowers' skill/dispatch patterns, CodeGraph's structural index, skill-cluster routing, and peon-ping voice packs — all of which run live on the author's machine every day. Every install surface here mirrors a corresponding private, machine-specific piece that stays referenced, never vendored (see UPSTREAM.md).

Objective

Give another operator a readable, reviewable installer for the same disciplined work loop, without requiring the author's private memory, credentials, or voice packs.

Users

Developers who want an ISA-driven (Ideal State Artifact) PAI-style loop in OpenCode, Cursor, or optionally Claude Code / Codex, with or without GSD/superpowers already installed.

Value

Safe, dry-runnable, backup-first install; optional surfaces stay opt-in; no forced subscription, auth, or specific model.

Metrics

ISA-tracked: 31/31 Ideal State Criteria currently satisfied and verified by ./verify.sh.

System Architecture (Layered)

Templates & Docs (data layer) templates/AGENTS.md, templates/cursor.*, templates/codex.AGENTS.md, templates/CLAUDE.md.template · docs/pai-flow.md (unified-flow source of truth), docs/skill-clusters.md, docs/peon-ping-packs.md, docs/codegraph-routing.md · docs/parallel-dispatch.md and docs/multi-surface-architecture.md are retired redirect stubs pointing at docs/pai-flow.md
Install Orchestration (processing layer) install.sh (flag parsing, env export) · scripts/install-*.sh (one script per surface) · scripts/lib.sh (say / ensure_dir / backup_file / install_file)
PAI-shell (always-on outer layer) The 7-phase Observe→Think→Plan→Build→Execute→Verify→Learn loop is always active as documented convention; enforcement is a future sub-project (B). The <temperance-context> block is the one always-on enrichment layer, emitted every prompt.
gsd-core-backbone (recommended-default workflow layer, still detect-only) gsd-core (open-gsd/gsd-core, npx @opengsd/gsd-core@latest) is the recommended default backbone: Discuss → Plan → Execute → Verify → Ship, commands in canonical /gsd-* hyphen form. It is recommended, not required — install-gsd.sh and --with-gsd remain detect-only and never vendor gsd-core. When gsd-core is absent, the flow degrades to the superpowers chain: brainstorming → writing-plans → subagent-driven-development → verification-before-completion → finishing-a-development-branch. The old ~/.claude/get-shit-done path (danielmiessler lineage) is deprecated/legacy; install-gsd.sh now recognizes gsd-core and points at the npx install.
skill-cluster-resolver (discovery / lazy-load layer) ~/.agents/skill-clusters/skill-index.json sits between "backbone names a skill" and "tool runs" — the context-economy mechanism. active-spoke skills are symlinked and loaded; deferred clusters are resolved on demand; archived skills are read from their indexed path; not-found is a hard error.
tool-inventory (services layer) superpowers skills (brainstorming, writing-plans, subagent-driven-development, dispatching-parallel-agents, verification-before-completion, finishing-a-development-branch) · temperance-parallel-dispatch (external backends command-code/grok/kimi via temperance-batch) · Pulse compatibility server (voice, --with-voice) · CodeGraph CLI (structural index)
Runtime Loop (output layer) Observe → Think → Plan → Build → Execute → Verify → Learn. The Execute phase invokes temperance-parallel-dispatch only when 2+ independent tasks would benefit from external backends; the full decision doctrine now lives in docs/pai-flow.md's Execute row (superpowers dispatch vs gsd-core execute-phase/workstreams), not docs/parallel-dispatch.md.
