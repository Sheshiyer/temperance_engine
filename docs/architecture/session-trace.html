Session Trace One concrete walkthrough, from installer flags to a finished Execute phase

Worked example. This traces a single real invocation: ./install.sh --with-claude --with-gsd --with-voice, followed by one live session that reaches a parallel-dispatch decision at Execute. Every step cites the exact file/function from system-internals.html.

Part A — Install

  1. Flags parsed. CLAUDE_MODE=install, GSD_MODE=install, VOICE_MODE=force (force needed unless on macOS); all others stay at their defaults (OpenCode/Cursor install, Codex skip).
  2. install-pai.sh runs. $HOME/AGENTS.md written. Because CLAUDE_MODE=install: $PAI_HOME/PAI/PULSE and $PAI_HOME/hooks created, CLAUDE.md.template and compat-server.ts copied in (each preceded by a timestamped backup of anything already at that path). If bun exists, any stale compat-server.pid process is killed and a fresh server is spawned on :31337.
  3. install-skill-clusters.sh runs. skill_cluster_resolver.mjs copied to $CODEX_HOME/hooks/ unconditionally. If $AGENTS_HOME/skill-clusters/package.json exists, the three verification commands are printed as a hint.
  4. install-peon-ping.sh runs. Not skipped (voice forced). Checks $PAI_HOME/hooks/peon-ping/peon.sh is executable. If it isn't yet (peon-ping wasn't installed separately), prints guidance and exits 0 — the install continues either way.
  5. install-codegraph.sh runs. If $AGENTS_HOME exists and codegraph is on PATH, runs codegraph init -i once (skipped if .codegraph already present).
  6. install-gsd.sh runs. GSD_MODE=install, so it checks $HOME/.claude/get-shit-done. On this machine it exists → prints "GSD detected" + a pointer to docs/parallel-dispatch.md. Zero files written.
  7. configure-opencode.sh runs. OpenCode and Cursor templates land in their default locations; Codex is skipped (not requested).
  8. verify-install.sh runs. All 21 required files present, all shell scripts lint clean, no private paths found, all content assertions pass → Temperance Engine verification passed. If any check had failed, install.sh would exit non-zero here — nothing after this point would print.
  9. Install flow complete. User is told to restart OpenCode/Cursor to reload instruction surfaces (Claude/Codex only if those optional surfaces were touched).

Part B — A Live Session Reaching Execute

  1. Session starts.setup If the user manually wired ParallelDispatchContext.hook.sh into their own ~/.claude/settings.json (it is not auto-registered), it checks .planning/ in the current project. Say this project has one, with 2 active workstreams and a config.json setting model_profile: "quality" — the hook injects a <system-reminder> noting all three (read-only) and pointing at docs/parallel-dispatch.md. It never reads PAI's steering rules or writes anything back to config.json — Temperance Engine's own preference store stays ISA.md alone.
  2. Observe.observe Agent reads live state. If it emits "Entering the Observe phase." and Pulse is running, compat-server.ts matches phase=observe → spawns peon.sh --pack glados --category task.acknowledge.
  3. Think.think Same pattern → pack hal_2001.
  4. Plan.plan Pack jarvis-mk2. If the work is GSD-plan-shaped, the agent may separately invoke /gsd:plan-phase here — that call is entirely external to Temperance Engine; nothing in this repo triggers or is triggered by it.
  5. Build.build Pack peon. Artifacts get created/edited.
  6. Execute — the parallel-dispatch decision point.execute Pack nier-2b. Before running the work, the agent consults docs/parallel-dispatch.md's decision tree: .planning/ exists and there are declared PLAN.md dependencies → GSD is installed (confirmed at install time) → use GSD execute-phase (wave-based, worktree-isolated). If instead the tasks were ad hoc and ephemeral with no shared state, the agent would use superpowers:dispatching-parallel-agents instead. If any task shared files/branch with another, everything runs sequentially regardless.
  7. Verify.verify Pack cortana. If this session were working on Temperance Engine itself, ./verify.sh would run here as the fresh-evidence check the Completion Rule requires.
  8. Learn.learn Pack sc_kerrigan. If this is an ISA-tracked project, decisions and verification evidence get appended to its ISA.md — exactly the pattern used for ISC-28..31 in this repo's own ISA.md.

Part C — Failure Modes, Traced

Missing pieceWhat actually happens
bun not installedPulse server files are copied but never started; any /notify POST simply has nowhere to land (connection refused). Session continues normally with no voice.
peon.sh missing or not executablecompat-server.ts still responds 200 to /notify with {played: false, reason: "bun-runtime-required"}-style fields; no sound, no error surfaced to the agent.
GSD not installed, --with-gsd passed anywayinstall-gsd.sh prints the not-found branch; at runtime, docs/parallel-dispatch.md's decision tree falls through to the superpowers-only branches since GSD-gated options require detecting ~/.claude/get-shit-done.
codegraph CLI not installedStructural search guidance in docs/codegraph-routing.md is simply unusable; no fallback is installed automatically (the doc says to use direct file reads/grep instead).
